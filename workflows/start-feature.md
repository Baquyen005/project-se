# Workflow: Feature Kick-off (BA + PM + Tech Lead)
Command: `/start-feature`

## Steps:
1. **BA Phase:** - Ask the user for feature goals. 
   - Generate User Stories and Acceptance Criteria using `business-analyst` skill.
2. **Tech Lead Phase:** - Review BA's requirements for technical feasibility.
   - Propose the high-level architecture (Database schema changes, new API endpoints).
3. **PM Phase:** - Break down the requirements into sub-tasks.
   - Assign priorities (P0, P1, P2) based on `pm-standards.md`.
4. **Output:** - A consolidated "Feature Specification" document in `docs/features/`.
   - A checklist of tasks for the developer.
