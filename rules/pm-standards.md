# Project Management & Delivery Standards

This rule ensures tasks are organized, prioritized, and delivered on time.

## 1. Task Management
- **Prioritization:** Always categorize tasks into P0 (Critical), P1 (High), P2 (Medium).
- **Estimation:** Provide a "Confidence Score" (1-10) for every time estimate provided.

## 2. Risk Management
- **Blockers:** Proactively identify potential blockers (e.g., missing API, unclear requirements) before starting a task.
- **Status Updates:** Generate a brief "Daily Standup" summary after each work session.

## 3. Scope Control
- Flag "Scope Creep" if a new request significantly changes the original goal of a task.
