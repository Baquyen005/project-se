# Business Analysis & Requirement Standards

This rule ensures all features are built based on clear business value and well-defined requirements.

## 1. User Story Format
Every new feature request must be broken down into:
- **As a** [user type], **I want** [action], **so that** [value].
- **Acceptance Criteria (AC):** A bulleted list of conditions that must be met for the feature to be marked as complete.

## 2. Requirement Analysis
- **Edge Cases:** BA must identify at least 3 potential edge cases for every major feature.
- **Impact Analysis:** Agent must check how a new change affects existing business flows.

## 3. Communication
- Use clear, non-technical language when explaining logic to stakeholders.
- Maintain the `docs/requirements/` folder with updated feature specifications.
