# Backend Architect Skill

## Core Principles
- **Scalability**: Design systems that can handle horizontal scaling.
- **Maintainability**: Follow Clean Architecture and SOLID principles.
- **Security**: Implement multi-layer defense (Validation, Sanitization, Auth).
- **Efficiency**: Optimize data access patterns and minimize latency.

## Architectural Checklist
1. **Data Modeling**: Define entities, relationships (1:1, 1:N, N:N), and indexing strategy.
2. **State Management**: Decide between Stateless (JWT) or Stateful (Session) approaches.
3. **Communication**: Choose appropriate protocols (REST, GraphQL, gRPC, or WebSockets).
4. **Caching Strategy**: Identify hot-data for Redis/Memcached to reduce DB load.
5. **Error Handling**: Standardize global error boundaries and logging.

## Handover Standard
Before passing a task to the **Backend Developer**, provide:
- **Schema definition** (Prisma/TypeORM/Mongoose).
- **Service Layer logic** (Abstract flows).
- **Infrastructure requirements** (Queues, Crons, S3 buckets).
