# 🧠 Project Memory System

This folder stores long-term project knowledge, architectural evolution, and past experiences to ensure continuity across different Agent sessions.

## 📋 Usage Guidelines
1. **Read Before Starting**: Every Agent must read `project-context.md` at the start of a task to understand the environment.
2. **Update After Task**: Upon completing a major workflow (e.g., `/feature-cycle`), the Agent must update `lessons-learned.md` if new insights were gained.
3. **Trace Decisions**: Consult `decision-log.md` before proposing major technical changes to understand previous constraints.
